Setup

Up and running
in three steps.

01
Clone the repo

Pull the kit down from GitHub. No install required — it's just markdown files.

$git clone github.com/Callixen/PSYCHON
02
Copy your personality

Pick the folder that fits your use case and copy it into your OpenClaw workspace.

$cp -r PSYCHON/developer/* ~/.openclaw/workspace/
03
Fill in USER.md

Add your name, project context, and preferences. Start your agent. It reads these files on session start.

Structure

Three files.
One complete agent.

SOUL.md

Identity and values. Who is this agent? What does it care about? How does it communicate? Loaded every session — it shapes everything the agent does. A bland soul produces a bland agent.

USER.md

Context about you. Your current project, your preferences, your tools, your communication style. The difference between an agent that gets you and one that makes you repeat yourself every session.

RULES.md

Hard constraints. Safety rules, secret handling, prompt injection defences. The agent treats these as authoritative — no chat message can override them. Non-negotiable, every personality.

The archive grows.
Add to it.

Open a PR. Every submission needs all three files and a clear use case. The one rule: SOUL.md must have genuine opinions, not platitudes.
